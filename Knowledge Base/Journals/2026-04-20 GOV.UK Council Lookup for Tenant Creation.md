# 2026-04-20 GOV.UK Council Lookup for Tenant Creation

**Phase:** 5 — MVP Build  
**Confidence Level:** High

---

## Summary

Added GOV.UK Local Authorities API integration to the platform admin tenant creation flow. Platform admins now look up a council by postcode rather than typing its name freehand, reducing data entry errors and ensuring council names/slugs are sourced from an authoritative register.

---

## Changes

### Backend — `src/routes/platform-admin.js`

New route: `GET /api/platform/council-lookup?postcode=`

- Proxies calls to `https://www.find-local-council.gov.uk/api/local-authority`
- 6-second `AbortController` timeout on all outbound fetches
- Handles GOV API's 301 redirect (single match) by fetching the authority record and normalising to `{ authorities: [...] }`
- Handles GOV API's 200 (multiple addresses) by fetching each authority slug concurrently
- **Error classification is explicit:**
  - `kind: "validation"` — bad postcode format, postcode not found (400/404 from GOV) — frontend shows inline error, no fallback
  - `kind: "service"` — timeout, network failure, unexpected status, malformed JSON — frontend shows fallback notice (502)
- All service-level failures are `console.error`'d for observability
- Route is platform-admin protected (requires valid platform admin session)

### Frontend — `frontend/src/pages/PlatformTenantCreatePage.jsx`

Full rewrite. State machine with five states:

| State | Meaning |
|---|---|
| `idle` | Postcode entry form shown |
| `loading` | GOV API call in flight |
| `select` | Multiple authorities returned — user picks one |
| `confirmed` | Single authority confirmed — detail form shown, fields pre-filled |
| `service_error` | GOV API unavailable — manual entry fallback with warning banner |

Behaviour:
- Single match → auto-selects and proceeds to detail form
- Multiple matches → user selects from list, then proceeds
- Service failure → warning banner + manual entry form (no postcode required)
- Validation error (invalid/unknown postcode) → inline error, stays on postcode step
- `toSlug()` derives slug/subdomain from authority name (`toLowerCase`, replace non-alphanumeric with `-`, trim to 63 chars)
- Slug and subdomain fields remain editable for override
- Subdomain hint shows live preview: `{subdomain}.zanflo.com`
- "Change" and "Start over" controls allow correction at any point

### API client — `frontend/src/api.js`

Added `councilLookup(postcode)` → `GET /api/platform/council-lookup?postcode=…` with `includeTenantHeader: false`.

## Architectural decisions

**Server-side proxy, not direct browser fetch.**  
The GOV API is unversioned, may change shape, and warns about rate limits. Proxying through the Worker gives a single point of control for timeouts, error normalisation, and any future caching or circuit-breaking without touching the frontend.

**Two distinct error kinds (validation vs service).**  
Conflating them would either block signup on GOV API outages (bad) or let through garbage postcode input (also bad). The explicit `kind` field lets the frontend treat them differently without parsing error strings.

**Slug derived from GOV authority slug, not from name.**  
The GOV API returns a `slug` field (e.g. `westminster`) which is already authority-canonical. When a single match is returned we use `toSlug(auth.name)` as the suggestion (consistent with multi-authority path where we only have names). Either way the admin can override before saving — backend uniqueness check remains authoritative.

---

## Risks / caveats

- GOV API is explicitly documented as unversioned and subject to change without notice. If its response shape changes, the `kind: "service"` path will activate and prompt manual entry — degraded but not broken.
- The multi-authority path fetches each authority record individually (N+1). In practice postcodes rarely span more than 2–3 authorities, so this is acceptable. Could be parallelised with `Promise.all` (it already is) and capped if needed.
- No caching of GOV responses. Could add short-lived KV caching later if rate limiting becomes a problem.

---

## Deferred

- Caching GOV API responses in Cloudflare KV (postcode → authority, short TTL)
- Circuit-breaker / exponential back-off if GOV API is persistently down
- Storing the GOV authority slug on the tenant record for future cross-referencing
